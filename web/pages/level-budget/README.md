# الميزانية - جدول التسوية (Levelling Calculation)

## 1) وظيفة الصفحة
الصفحة تطبق **طريقة H.I (Height of Instrument)** لحساب مناسيب النقاط من:
- `BS` (Back Sight)
- `IS` (Inter Sight)
- `FS` (Fore Sight)

وتُخرج:
- `H.I / H.L.S`
- `RL / P.H`
- فرق المناسيب
- حالة التحقق الحسابي (`ΣBS - ΣFS` مقابل `Last RL - First RL`)
- `Misclosure` و `Error/Set-up` عند إدخال منسوب نهاية مرجعي.

## 2) مدخلات أعلى الصفحة
- بيانات وصفية: `Job Title`, `Job No`, `Date`, `Observer`, `Staffman`.
- `Start Value`: منسوب البداية.
- `Finish Value`: منسوب النهاية المرجعي (اختياري، لكنه مطلوب لخطأ قفل صحيح).
- `Tolerance`: سماحية الخطأ.

### نواتج أعلى الصفحة
- `Misclosure = Computed Last RL - Finish Value`
  - إذا `Finish Value` فارغ: تعرض الصفحة `Last RL` فقط كقيمة مرجعية.
- `Set-ups`: عدد مرات وجود `BS` (نقاط نقل الجهاز/Turning Points).
- `Error/Set-up = Misclosure / Set-ups`.
- `Diff first-last = Last RL - First RL`.
- `Diff ΣBS-ΣFS = ΣBS - ΣFS`.

## 3) الأعمدة الأساسية في الجدول
- `S.No / Point`: اسم/رقم النقطة (إدخال).
- `Back Sight (C)`: BS (إدخال).
- `Inter Sight (D)`: IS (إدخال).
- `Fore Sight (E)`: FS (إدخال).
- `H.L.S (F)`: ناتج H.I.
- `P.H (G)`: ناتج RL.
- `Point No (H)`: ترقيم/اسم نقطة تلقائي.
- `DEF (+)(-) HLS (I)`: فرق منسوب بالنسبة للمرجع داخل الوقفة.
- `DEF TO PO (J)`: فرق مبني على قراءة الصف السابق/الحالي.
- `NOTE (K)`: ملاحظة مثل `Turning Point 1` أو `end`.

## 4) الأعمدة المساعدة (O → AC)
معروضة داخل قسم قابل للفتح:
- `O`: H.I المحسوب.
- `P`: RL المحسوب.
- `R`: H.I الجاري داخل نفس الوقفة.
- `S`: BS الفعّال داخل الوقفة.
- `T`: مرجع RL للوقفة (`R - S`).
- `U`: الفرق (`P - T`).
- `V`: تصنيف (`CUT` أو `FALL`).
- `W`: هل الصف `Turning Point`.
- `X`: عداد داخلي.
- `Y`: رقم Turning Point.
- `Z`: النص النهائي للملاحظة.
- `AA`: إجمالي عدد Turning Points.
- `AB`: ترتيب TP عند صف فيه FS.
- `AC`: علامة نهاية (`end`) عند تحقق شرط نهاية المسار.

## 5) المعادلات الأساسية المطبقة
- بداية الوقفة:
  - `H.I = RL + BS (+ IS إذا كانت موجودة في نفس الصف حسب تنسيق الشيت)`
- لكل صف بعده:
  - `RL = H.I(previous) - IS - FS`
  - `H.I(current) = RL + BS + IS`
- المرجع داخل الوقفة:
  - `R = BS ? H.I : R(previous)`
  - `S = BS ? BS : S(previous)`
  - `T = R - S`
  - `U = RL - T`
  - `V = U>0 ? CUT : U<0 ? FALL : ""`
- التحقق:
  - `ΣBS - ΣFS`
  - `Last RL - First RL`
  - `Closing Error = |(ΣBS-ΣFS) - (LastRL-FirstRL)|`

## 6) التطبيق العملي
عند إدخال الصفوف بنفس ترتيب ملف الإكسل:
- صف بداية يحتوي `BS` يولد `Turning Point`.
- صفوف `IS` تعطي نقاط بينية على نفس H.I.
- صف `FS + BS` يعمل كصف انتقال جهاز:
  - يحسب RL من H.I القديم باستخدام FS.
  - ثم يفتح H.I جديد من RL الناتج باستخدام BS.

## 7) ملاحظات مهمة
- الخلايا الصفراء في الصفحة هي إدخال المستخدم.
- إذا `Finish Value` غير مدخل:
  - `Misclosure` و `Error/Set-up` لا يمثلان خطأ قفل نهائي هندسي.
- التوافق:
  - تم تجنب الاعتماد على `IFS` داخل الصيغ النصية؛ المنطق مطبق مباشرة في JavaScript.

## 8) الاستخدام السريع
1. أدخل `Start Value` والقراءات (`BS/IS/FS`) وأسماء النقاط.
2. أدخل `Finish Value` إن كان لديك منسوب مرجعي نهائي.
3. اضغط **احسب وحقق**.
4. راجع:
   - النتائج الأساسية في الجدول.
   - التحقق الحسابي.
   - الأعمدة المساعدة من القسم القابل للفتح.
